Traffic Citation and Reporting System
The Traffic Citation and Reporting System is a web-based application designed to assist police officers in efficiently creating and managing citations for traffic violations.

User Roles
The system supports two types of users:

Administrator:
Responsible for managing system users. Administrators can create, modify, and delete user accounts.
Local User (Officer):
Designated for field officers who issue traffic citations. Officers can perform the following functions:
Create, modify, and delete traffic citations
Create, modify, and delete driver information
Create, modify, and delete vehicle information
Search citations, drivers, and vehicles
Generate detailed reports with advanced filtering capabilities (by date range, issuing officer, driver’s license number, vehicle plate number, and other criteria)

Officer Dashboard
Upon login, officers are presented with a comprehensive dashboard displaying three key tables:
Recent or pending citations
Driver records
Vehicle records  
along with relevant associated information.


Traffic Violations
The system supports three categories of traffic violations:
Parking violations
Moving vehicle violations
Fix-it violations
For moving vehicle violations, the system automatically prompts the officer to issue a mandatory traffic school requirement. The officer must specify a traffic school duration of more than 8 hours, which is included in the citation. The driver is required to attend the assigned traffic school.

Data Validation
The system enforces strict validation rules and rejects any submitted information that does not meet the defined requirements.

Administrator Dashboard
Administrators have access to a dedicated dashboard that displays a complete list of all system users along with their details. Administrators can create, modify, and delete user accounts. Each user account includes the following information:
Username
Account Type (Admin or Local)
Account Status (Active or Inactive)
Password
Full Name
The system also enforces data validation for all administrative actions and rejects entries that do not comply with the defined requirements.
